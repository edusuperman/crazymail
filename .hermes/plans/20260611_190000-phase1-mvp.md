# 阶段一：产品 MVP 实施计划

> **战略指引：** 先读 STRATEGY.md，确保所有实现与终极目标对齐。
> **时间策略：** 不锁定时间，全速推进。

**目标：** 上线一个能用的临时邮箱网站，用户能生成邮箱、接收邮件、查看内容。

**验收标准：**
- 用户能打开网站 → 生成临时邮箱 → 收到邮件 → 查看内容
- 网站可被 Google 收录（SSR + sitemap）

**技术栈：**
- 前端：Next.js 15 + Tailwind CSS
- 后端：FastAPI 或 Next.js API Routes
- 邮件 API：Mail.tm / TemporaryMail
- 数据库：Supabase
- 部署：Vercel 或 Cloudflare Pages

---

## Task 1: 邮件 API 调研与选型

**Objective:** 确定首选邮件 API，测试连通性

**步骤:**
1. 测试 Mail.tm API（评分最高，延迟<5秒）
2. 测试 TemporaryMail API
3. 测试 Temp-Mail.io API
4. 选择最稳定的作为首选

**验收:** 至少一个 API 能成功创建邮箱并接收邮件

---

## Task 2: 创建邮件 API 适配器

**Objective:** 封装邮件 API 调用，支持多接口切换

**Files:**
- Create: `backend/services/email_adapter.py`

**功能:**
- 创建临时邮箱
- 查询收件箱
- 获取邮件详情
- 接口健康检查

---

## Task 3: 创建 FastAPI 邮件接口

**Objective:** 提供邮件相关的 API 端点

**Files:**
- Create: `backend/routers/email.py`

**接口:**
- `POST /api/v1/email/create` - 创建临时邮箱
- `GET /api/v1/email/{email_id}/messages` - 获取收件箱
- `GET /api/v1/email/messages/{message_id}` - 获取邮件详情

## Task 9: 端到端测试

**Objective:** 验证完整流程

**测试场景:**
1. 打开网站 → 生成邮箱 → 复制地址
2. 向该邮箱发送邮件 → 刷新收件箱 → 查看邮件
3. 邮件过期后自动清理

---

## 风险和注意事项

1. **邮件 API 稳定性**: 可能有频率限制或服务不稳定，需要实现多接口切换
2. **国内网络**: 访问国外 API 可能不稳定，考虑使用代理
3. **域名备案**: 如果面向国内用户，可能需要备案
4. **AdSense 审核**: 需要足够内容和流量才能通过审核
