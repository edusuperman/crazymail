# Definition of Done（DoD）
# 版本：v1.0 | 2026-06-11
# 所有任务必须同时满足以下条件才算完成

## ════════════════════════════════════════════
## 通用 DoD（每个任务必须满足）
## ════════════════════════════════════════════

- [ ] 代码能正常运行，无语法/导入错误
- [ ] 通过所有相关测试（单元测试 + 手动验证）
- [ ] 符合 AGENTS.md 中的代码规范
- [ ] 符合 STRATEGY.md 当前阶段目标
- [ ] 无硬编码（密钥、URL 等走环境变量）
- [ ] 相关文档已更新（如果适用）
- [ ] Hermes 审查通过
- [ ] 用户最终确认（关键任务）

## ════════════════════════════════════════════
## 特定类型 DoD
## ════════════════════════════════════════════

### 新功能
- [ ] 有用户故事验收标准
- [ ] 有对应的测试用例
- [ ] 功能可手动验证

### 重构
- [ ] 不改变外部行为
- [ ] 测试通过率不变或提升
- [ ] 代码可读性提升

### Bug 修复
- [ ] 复现步骤已验证不再出现
- [ ] 有回归测试防止再次发生
- [ ] 修复范围最小化

### 前端页面
- [ ] SSR 正常工作
- [ ] Meta tags 完整
- [ ] 移动端响应式
- [ ] Core Web Vitals 达标

### API 接口
- [ ] 有请求/响应文档
- [ ] 有错误处理
- [ ] 有输入校验

## ════════════════════════════════════════════
## 任务粒度指南
## ════════════════════════════════════════════

**推荐：30-90 分钟可完成**

- 太大（>2 小时）：拆分为多个子任务
- 太小（<10 分钟）：合并到相关任务中
- 一个任务 = 一个清晰的交付物

## ════════════════════════════════════════════
## DoD 检查流程
## ════════════════════════════════════════════

1. OpenCode 完成任务后，自查通用 DoD
2. Hermes 对照 DoD 审查
3. 不通过 → 打回重做，说明具体哪项不达标
4. 通过 → 提交用户确认
5. 用户确认 → 任务完成，进入下一个
