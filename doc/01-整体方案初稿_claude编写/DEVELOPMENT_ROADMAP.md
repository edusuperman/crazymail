# CrazyMail 开发路线图

## 总体原则

- 每个 Phase 必须有明确的「完成标准」，达到后再推进
- 可视化（PixiJS）永远是最后做的，不影响核心功能开发
- 任何模块完成后，立即补充对应的「人工替代方案」文档

---

## Phase 0：环境准备（Day 1-3）

**目标**：开发环境搭通，不写业务代码

### 任务清单
- [ ] 安装 Python 3.12.4 + uv
- [ ] 安装 Node.js 22 LTS + pnpm
- [ ] 安装 Docker Desktop，启动 PostgreSQL + Redis
- [ ] 安装 Ollama，下载 qwen2.5:7b 和 deepseek-r1:8b
- [ ] 安装 Cursor（配置 .cursorrules）
- [ ] 配置 Supabase 项目（免费层）
- [ ] 申请 Claude API Key + DeepSeek API Key
- [ ] 创建项目 Git 仓库（私有）
- [ ] 初始化 FastAPI 项目骨架（main.py 能跑通）
- [ ] 初始化 Next.js 15 项目（首页能渲染）
- [ ] 运行数据库迁移（创建所有表）

**完成标准**：`curl http://localhost:8000/health` 返回 200，前端 localhost:3000 能访问

## Phase 1：单站点内容流水线（Week 1-3）

**目标**：跑通一个站点的完整7阶段，产出一篇通过质检的文章

### Week 1：后端 Agent 骨架
- [ ] CrewAI 定义李纲（关键词研究）Agent
- [ ] CrewAI 定义张择端（SERP+大纲）Agent
- [ ] CrewAI 定义李清照（写作）Agent
- [ ] LangGraph Workflow：串联前3个阶段
- [ ] FastAPI Router：`POST /api/v1/content/tasks` 接口

### Week 2：质检 + 完整流水线
- [ ] 岳飞质检节点（集成 Originality.ai API + 自定义评分）
- [ ] 种师道 SEO 优化节点
- [ ] 韩世忠链接建设节点
- [ ] 宋徽宗 Human-in-the-Loop 节点（`interrupt_before`）
- [ ] 完整 LangGraph Workflow 7阶段串通
- [ ] Dashboard 审核界面（基础版，不需要动效）

### Week 3：调试 + 验证
- [ ] 端到端测试：输入关键词 → 输出已发布文章
- [ ] 质检分数校准（调整各项阈值）
- [ ] 去AI味策略测试（Originality.ai 得分 < 30）
- [ ] 第一篇文章发布到真实站点

**完成标准**：
- 对任意英文关键词，24小时内自动产出一篇文章
- AI检测分 < 30，SEO分 > 75
- 宋徽宗审核界面可以通过/打回文章

## Phase 2：多站点扩展 + 数据层（Week 4-6）

**目标**：扩展到3个站点，建立基础监控

### 任务清单

## 关键决策节点

| 时间 | 决策内容 |
|------|---------|
| Phase 1 结束 | 内容质量是否达标？是否继续扩站？ |
| Phase 2 结束 | 哪个 Niche 的 RPM 最高？第二批主推该方向 |
| Phase 4 第30天 | 好汉账号是否安全稳定？是否扩充到20个？ |
| Phase 5 结束 | 可视化 Demo 是否有展示/融资价值？ |
