## Stage 2: 大纲设计
## ════════════════════════════════════════════

### 2.1 文章结构要求

```
标题（含关键词，50-60字符）
├── 引言（100-150字，直接回答用户问题）
├── H2: 主体段落 1
│   ├── 正文（200-300字）
│   └── 小结（1-2句）
├── H2: 主体段落 2
│   ├── 列表/表格（数据支撑）
│   └── 正文（200-300字）
├── H2: 主体段落 3
│   ├── 个人观点/经验
│   └── 正文（200-300字）
├── H2: FAQ（3-5个问题）
├── H2: 总结/CTA
└── 作者 Bio
```

### 2.2 排版要求

- 段落长度：3-5句话，不超过100字
- 使用 H2/H3 标题结构
- 列表和表格突出关键信息
- 粗体强调重要观点
- 适当使用引用块
- 图片 alt 文本（如适用）

## Stage 3: 写作 + 人味注入
## ════════════════════════════════════════════

### 3.1 人味注入清单

| 维度 | 要求 | 示例 |
|------|------|------|
| **口头禅** | 2-3个标志性表达 | "Look," / "Here's the thing" / "I've tested this" |
| **轻微错误** | 偶尔用 em dash，偶尔首字母不大写 | "it's — not perfect" |
| **个人观点** | 每篇1-2处 | "In my testing..." / "I've found that..." |
| **数字随机化** | "3 ways" vs "three ways" | 随机选择 |
| **段落节奏** | 偶尔单句段落 | "That's it." |
| **转折词多样性** | 不重复使用 | "however" / "but" / "that said" / "still" |
| **第一手经验** | 提及亲身测试 | "When I tried this..." |

### 3.2 去AI味检查点

- ❌ 不要："In conclusion," / "Furthermore," / "Moreover,"
- ❌ 不要：过度使用 "It's important to note that..."
- ❌ 不要：段落开头都是 "First," / "Second," / "Third,"
- ✅ 要：自然过渡，像朋友聊天

### 3.3 写作 Prompt 模板

```
你是一个有5年经验的隐私保护博主。请用以下风格写文章：

1. 像朋友聊天一样自然
2. 分享你的第一手测试经验
3. 偶尔用口语化表达（"Look," / "Here's the thing"）
4. 段落不要太长（3-5句话）
5. 每篇至少1处个人观点（"In my testing..."）
6. 不要用 "Furthermore" / "Moreover" / "In conclusion"
7. 使用列表和表格让内容更易读

关键词：[填入]
大纲：[填入]
```

### 4.1 质检清单

| 检查项 | 标准 | 工具 |
