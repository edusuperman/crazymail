# 邮箱网站矩阵开发闭环体系

## 总体架构

```
API探查 → Prompt生成 → 网站制作 → 操作手册 → 自动化测试 → 问题修复 → 知识沉淀 → Skills整合
   ↓         ↓          ↓          ↓           ↓           ↓          ↓          ↓
 标准流程    框架模板    Lovable    拟人化手册    全面测试    循环修复    项目+公共    闭环
```

## 九大环节

### 环节1：API探查与学习
目标：对提供API的邮箱服务进行全面、系统、深入的探查
产出：API探查报告

### 环节2：Prompt生成
目标：结合API探查结果，生成给Lovable的结构化Prompt
产出：标准化Prompt文档

### 环节3：网站制作
目标：用Lovable生成前端，OpenCode开发后端
产出：可运行的网站

### 环节4：操作手册编写
目标：基于API和原站功能，编写拟人化操作手册
产出：详细操作手册

### 环节5：自动化测试
目标：按操作手册进行全面自动化测试
产出：测试报告

### 环节6：问题修复
目标：测试不通过的问题记录、分析、修复
产出：修复后的网站

### 环节7：知识沉淀
目标：将过程中的经验沉淀到项目知识库和公共方法论
产出：更新的知识库

### 环节8：Skills整合
目标：形成可复用的skills，与已有skills库整合
产出：增强的skills库

### 环节9：闭环验证
目标：确保整个体系运转良好
产出：可交付的网站
