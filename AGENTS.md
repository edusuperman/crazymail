## ════════════════════════════════════════════
## 终极目标（不可动摇）
## ════════════════════════════════════════════

**靠临时邮箱网站矩阵赚钱。**

当前阶段：阶段一（产品 MVP）
- 核心功能：临时邮箱的生成、接收、查看、过期
- 技术栈：Next.js 15 + FastAPI + 邮件 API
- 目标：上线一个能用的产品

**不要做阶段二/三的功能（内容工厂、108好汉、皇城司等），除非明确要求。**

## ════════════════════════════════════════════
## 技术栈（锁定）
## ════════════════════════════════════════════

**前端：**
- Next.js 15 (App Router) + TypeScript
- Tailwind CSS
- 响应式设计（移动端优先）

**后端：**
- Python 3.12 + FastAPI
- uv 管理依赖（不用 pip）
- Pydantic 数据校验

**数据库：**
- Supabase（PostgreSQL）
- 使用 Supabase Python 客户端（不用直接 SQL）

**邮件 API：**
- Mail.tm / TemporaryMail / Temp-Mail.io
- 多接口支持，可切换

**部署：**
- Vercel 或 Cloudflare Pages

## ════════════════════════════════════════════
## 代码规范（严格遵守）
## ════════════════════════════════════════════

**Python：**
- 3.12+，类型注解必须完整
- uv 管理依赖，不用 pip
- snake_case 命名
- 核心逻辑中文注释

**TypeScript：**
- 严格模式，禁止 any
- kebab-case 文件名，PascalCase 组件名
- 组件拆分合理，单一职责

**通用：**
- 所有配置通过环境变量注入
- 禁止硬编码密钥/URL/账号
- 每个函数/组件必须有类型定义

## ════════════════════════════════════════════
## SEO/GEO 规范（从第一天开始）
## ════════════════════════════════════════════

**每个页面必须包含：**
- SSR（服务端渲染）— 不用纯客户端渲染
- 语义化 HTML（h1, h2, article, nav 等）
- Meta tags（title, description, og:image 等）
- Schema.org 结构化数据（JSON-LD）
- 干净的 URL 结构（/inbox/[id]，不是 ?id=xxx）
- 内部链接策略
- 图片 alt 文本
- Core Web Vitals 优化（LCP, CLS, INP）

**GEO（生成式引擎优化）：**
- 内容结构清晰，便于 AI 引用
- 包含权威数据和来源
- FAQ 格式内容
- 直接回答问题的格式
